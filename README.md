## Adding a Skill

1. Fork this repo and create a branch.
2. Create a folder named for your skill's slug:

   ```
   your-skill-name/
   ├── SKILL.md       # required: name + description frontmatter + agent instructions
   ├── catalog.json   # required: how it shows up + installs in the Bankr catalog
   ├── logo.svg       # recommended: square brand mark (png ok) — omit → provider initial
   ├── references/    # optional: supporting docs
   └── scripts/       # optional: helper scripts
   ```

3. Add `catalog.json` — it must validate and its `slug` must equal the folder name:

   ```jsonc
   {
     "schemaVersion": 1,
     "slug": "your-skill-name",
     "provider": "Your Brand",
     "providerUrl": "https://your.site",
     "logo": "logo.svg",
     "demo": { "title": "demo.sh", "language": "bash", "code": "..." },
     "setup": ["Step one", "Step two"],
     "install": {
       "type": "bankr",
       "repoPath": "your-skill-name",
       "command": "install the your-skill-name skill from https://github.com/BankrBot/skills/tree/main/your-skill-name"
     }
   }
   ```

   Skills that install from elsewhere use the external form instead:
   `"install": { "type": "external", "provider": "Acme", "command": "npx skills add acme/skills", "reference": null }`

4. Open a pull request describing what your skill does.

> **Note:** a folder without a valid `catalog.json` is skipped — it won't appear in the Bankr Discover catalog.

**Guidelines:** Keep `SKILL.md` clear and well-documented (`name` + `description` frontmatter are the single source for those fields). Include usage examples. Test before submitting.
